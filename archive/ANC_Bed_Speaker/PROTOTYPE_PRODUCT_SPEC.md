# 시제품 사양서

2단계: 임베디드 제품

이 문서는 수면용 ANC 제품의 임베디드 제품화 목표를 정리한 사양서다.
현재 활성 구현은 `/home/admin/anc.c` 기반 C 런타임이지만, 제품 요구사항과 장기 방향은 이 문서를 기준으로 다시 가져갈 수 있다.

## 목적
- PC 기반 프로토타입을 소형 임베디드 “스마트 베개”로 전환.
- 귀 위치 소음 상쇄 성능을 유지하면서 크기/전력 최적화.

## 사용 시나리오
- 가정용 침구 환경.
- 단일 사용자 귀 위치에서 코골이 상쇄.

## 성공 기준
- 1단계와 동일 또는 더 나은 성능:
  - 100-300 Hz 10-15 dB 이상 감소.
  - 말소리 인식도 50% 이하.
  - 지연 5 ms 이하.
- 배터리 사용 시간 6시간 이상.
- 안전한 음압 출력 (과도한 SPL 방지).

## 비목표
- 방 전체 소음 제거.
- 다중 사용자 동시 상쇄.
- 고출력 스피커 기반 오디오 재생 기능.

## 시스템 아키텍처

### 핵심 구성
- DSP SoC: STM32H7 또는 Raspberry CM.
- I2S MEMS 마이크 어레이 (기준 2 + 에러 1).
- 스피커 어레이 (베개 내장).
- 배터리 및 전원 관리.
- Bluetooth (설정/업데이트).

### 신호 흐름
1) I2S 마이크 입력 수집.
2) DSP에서 FxLMS 실행.
3) DAC/앰프를 통해 스피커 출력.
4) 에러 마이크 피드백으로 적응 업데이트.

## 하드웨어 요구

### 마이크
- I2S MEMS 마이크 어레이.
- 감도/위상 특성이 유사한 모델 권장.

### 스피커
- 3-5W 소형 스피커 4개.
- 베개 내부 구조와 일체화.

### 연산 장치
- STM32H7 (저전력, 실시간) 또는 Raspberry CM (개발 용이).
- DMA 기반 오디오 처리.

### 전원
- Li-ion 배터리 (3.7V 또는 7.4V).
- 6시간 이상 사용 가능한 용량.
- USB-C 충전.

### 통신
- BLE: 설정, 펌웨어 업데이트.
- 선택: USB 서비스 포트.

## 기구/외형
- 소프트 커버 내부 모듈형 구조.
- 에러 마이크 구멍(귀 옆 위치) 확보.
- 내부 공진 방지를 위한 흡음재.
- 발열 분산 설계.

## 소프트웨어/펌웨어

### DSP 코어
- C/C++ 구현 FxLMS.
- 고정소수점 또는 부동소수점 최적화.
- Secondary path는 부팅 시 또는 캘리브레이션 시 학습.

### 오디오 파이프라인
- I2S 캡처 + 더블 버퍼.
- 저지연 DSP 루프 (총 5 ms 이하).
- DAC 또는 I2S 앰프 출력.

### 설정
- BLE 앱 또는 간단 버튼 UI.
- 베개 두께/위치에 맞춘 프리셋.

### 안전 기능
- 출력 리미터.
- 워치독 및 안전 모드.

### 캘리브레이션/자가진단
- 전원 켜짐 시 마이크/스피커 체크.
- 선택: 조용한 환경에서 자동 캘리브레이션.

## 검증

### 음향 테스트
- 1단계와 동일한 프로토콜.
- 베개 내장 환경에서 성능 확인.

### 전원 테스트
- 실제 사용 조건에서 런타임 측정.
- 발열 검증.

### 신뢰성
- 8시간 연속 구동.
- 움직임/진동 내성 테스트.

## 문서 산출물
- 시스템 블록 다이어그램.
- 회로도 및 PCB 노트(커스텀 보드 시).
- 펌웨어 구조 문서.
- 성능 측정 리포트.

## 2단계 로드맵
- M1: 부품 선정 + 브레드보드 구성
- M2: 임베디드 FxLMS 동작
- M3: 베개 내부 통합
- M4: 성능 검증 및 튜닝

## 리스크/대응
- 지연 초과: 버퍼 축소, DMA/ISR 최적화.
- 전력 과다: 샘플레이트 조정, 연산 최적화.
- 공진 발생: 구조 변경 및 흡음재 추가.
- 마이크 간섭: 물리적 분리, EQ 보정.
