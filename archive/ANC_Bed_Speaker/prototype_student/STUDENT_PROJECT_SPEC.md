# 학생 프로젝트 사양서

1단계: PC 프로토타입

이 문서는 초기 Python 기반 ANC 실험의 기술 사양서다.
현재 활성 구현은 `/home/admin/anc.c` 기반 C 런타임이지만, stage 1 목표와 측정 기준은 이 문서를 보면 된다.

## 목적
- 코골이 소리에 대한 능동 소음 제어(ANC) 프로토타입을 실제로 동작시키는 것.
- 목표: 한 사람의 귀 위치에서 100-300 Hz 코골이 성분 10-15 dB 이상 감소.
- 말소리(300 Hz 이상) 인식도는 50% 이상 유지.
- 전체 지연 5 ms 이하.

## 사용 시나리오
- A(코골이) 소리가 발생.
- B(청자)는 베개에 눕고, 귀 옆에 에러 마이크를 배치.
- 시스템은 B의 귀 위치에서 코골이를 상쇄.

## 성공 기준
- 100-300 Hz 대역 10-15 dB 감소.
- 말소리 인식도 50% 이상.
- 전체 지연 5 ms 이하(AD/DA + DSP + 버퍼).
- 체감: “상대 코골이가 거의 안 들림.”

## 비목표
- 방 전체 소음 제거.
- 다수 사용자/다중 귀 위치 동시 상쇄.
- 제품화 형태나 배터리 구동.

## 시스템 개요

### 센서/액추에이터
- 기준 마이크: MEMS 마이크 1개 (Mic1, 코골이 방향).
- 에러 마이크: 1개 (귀 옆, 베개 표면).
- 스피커: 3-5W 소형 스피커 4개 (좌/우, 머리 양쪽).

### 연산 장치
- 노트북 또는 라즈베리 파이.
- 지연이 낮은 USB 오디오 인터페이스.

### 신호 흐름
1) Mic1으로 코골이 신호 측정.
2) DSP가 귀 위치 도달 신호 예측.
3) 역위상 신호 생성.
4) 스피커 출력.
5) 에러 마이크가 잔여 소음 측정.
6) FxLMS가 필터를 실시간 업데이트.

## 신호 정의(수학 모델)
- x[n]: Mic1 신호 (기준 신호)
- y[n]: 스피커 출력(역위상)
- d[n]: 귀 위치에서의 실제 코골이
- e[n]: 에러 마이크(귀 옆) 신호
- S(z): 스피커 → 귀 전달 경로(secondary path)
- W(z): 적응 필터

### 관계식
- e[n] = d[n] + S(z) * y[n]
- y[n] = W(z) * x[n]

### 목표
- E[e[n]^2] 최소화

## 알고리즘
- FxLMS (Filtered-x Least Mean Square).
- secondary path S(z)를 사전 추정한 S_hat(z)로 보정.
- 필터 입력: x_f[n] = S_hat(z) * x[n]
- 업데이트: W[n+1] = W[n] - μ * e[n] * x_f[n]
- 옵션: NLMS 적용
  - W[n+1] = W[n] - μ * e[n] * x_f[n] / (ε + ||x_f[n]||^2)

## Secondary Path 학습(필수)
1) ANC OFF.
2) 스피커에 white noise 출력(-12 dBFS 수준).
3) 에러 마이크로 수신.
4) LMS/NLMS로 S_hat(z) 추정.
5) 수렴 확인 후 S_hat(z) 고정.

## DSP 파라미터(초기 목표)
- 샘플레이트: 48 kHz 고정.
- 프레임 크기: 64 샘플 권장(1.33 ms). 128 샘플은 2.67 ms.
- 적응 필터 길이: W(z) 256 탭 권장(최대 512).
- S_hat(z) 길이: 256 탭 권장.
- 학습률(mu): 1e-4 ~ 1e-3 범위에서 안정성 기준으로 튜닝.
- 전처리: 100-300 Hz band-pass, 입력 정규화.
- 출력 보호: 소프트 리미터(예: ±0.8).

## 채널 매핑
- 입력: CH0=Mic1, CH1=Error Mic
- 기준 신호: x[n]=CH0
- 출력: CH0=Left Spk, CH1=Right Spk (또는 4채널)

## 하드웨어 배치

### 배치 예시(상단 뷰)
[코골이]
  소리 ->
  [Mic1]

   [Spk1]  [청자]  [Spk2]
           ^
       [에러 마이크]

### 주의 사항
- 에러 마이크는 귀 위치와 최대한 근접하게 배치.
- 스피커는 좌우 대칭 배치로 위상 편차 최소화.
- 스피커-에러 마이크 간 거리 기록(secondary path 정확도 중요).

## BOM(하드웨어 목록)
- MEMS 마이크 2개 (기준 1 + 에러 1)
- 3-5W 소형 스피커 4개
- USB 오디오 인터페이스(멀티채널, 저지연)
- 노트북 또는 라즈베리 파이
- 배선, 고정용 부품, 흡음재

## 소프트웨어 스택
- Python 3.10+
- sounddevice
- NumPy
- (선택) SciPy

## 인터페이스 요구
- 오디오 인터페이스는 최소:
  - 입력 2채널( Mic1/에러 마이크 )
  - 출력 2채널(Spk1/Spk2) 또는 4채널
  - 저지연 버퍼 설정 지원

## 지연 예산
- AD/DA: <= 2 ms
- DSP 계산: <= 1 ms (프레임 64 기준)
- 버퍼링: <= 2 ms
- 총합: <= 5 ms

## 측정/검증

### 테스트 신호
- 핑크노이즈 (기본 성능 점검)
- 실제 코골이 녹음 샘플
- 말소리 샘플(대역 유지 확인)

### 측정 지표
- 귀 위치 SPL 변화 (100-300 Hz 대역)
- 지연 시간 (임펄스 응답 측정)
- 말소리 인식도 테스트
- 안정성(10분 연속 구동 시 발산 없음)

### 절차
1) 마이크 감도 캘리브레이션.
2) Secondary Path 학습(ANC OFF).
3) 에러 마이크 기준 베이스라인 SPL 측정(ANC OFF).
4) FxLMS 활성화 후 SPL 재측정(ANC ON).
5) 100-300 Hz 대역 감소량 계산(Welch PSD 권장).
6) 말소리 인식도 유지 확인.
7) 10분 이상 안정성 확인.

## 로그
- 원시 마이크 신호 저장.
- 에러 신호 및 필터 계수 저장.
- S_hat(z) 임펄스 응답 저장.

## 리스크/대응
- 발산: 학습률 감소, 필터 길이 조정, secondary path 모델 안정화.
- 지연 초과: 버퍼 축소, 오디오 인터페이스 교체.
- 효과 부족: 배치 개선, 위상 보정.
- 출력 과포화: 리미터 강도 증가, 입력 스케일링.
