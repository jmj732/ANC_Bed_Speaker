# 코골이 소리 줄이기 프로젝트 - 쉬운 설명서

이 문서는 `prototype_student/` 기준의 초기 Python 프로토타입을 쉽게 설명하기 위한 문서입니다.
현재 활성 구현은 `/home/admin/anc.c` 기반 C 런타임이지만, 프로젝트 개념을 빠르게 이해할 때는 이 문서가 가장 읽기 쉽습니다.

## 한눈에 보는 프로젝트

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   코골이 소리  ───▶  마이크로 감지  ───▶  컴퓨터가 분석    │
│                                                             │
│                              ▼                              │
│                                                             │
│   귀에서 소음 감소  ◀───  스피커로 반대 소리 출력          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

---

## 1. 이 프로젝트는 무엇인가요?

### 간단한 설명
**옆 사람의 코골이 소리를 줄여주는 시스템**입니다.

노이즈 캔슬링 이어폰처럼, 코골이 소리를 감지해서 그 소리를 "상쇄"하는 반대 소리를 만들어 냅니다.
결과적으로 청자(잠을 자는 사람)의 귀에서 코골이가 훨씬 작게 들리게 됩니다.

### 비유로 이해하기

**물결 비유**를 생각해보세요:
- 연못에 돌을 던지면 물결이 퍼져나갑니다
- 만약 그 물결과 **정확히 반대되는 물결**을 만들면?
- 두 물결이 만나서 **서로 상쇄**되어 잔잔해집니다

소리도 물결(파동)입니다. 이 프로젝트는:
1. 코골이 소리(물결)를 감지하고
2. 정확히 반대되는 소리(반대 물결)를 만들어서
3. 두 소리가 만나면 서로 상쇄되게 합니다

```
원래 코골이:    ∿∿∿∿∿  (파동 모양)
반대 소리:      ∼∼∼∼∼  (위아래가 반대인 파동)
합쳐진 결과:    ━━━━━  (거의 평평 = 소리 감소!)
```

---

## 2. 왜 만들었나요?

### 해결하고 싶은 문제
- **배우자/동거인의 코골이** 때문에 잠을 못 자는 상황
- 기존 해결책(귀마개, 방 분리, 수술 등)의 불편함
- 노이즈 캔슬링 이어폰은 **귀에 끼고 자야 해서 불편함**

### 이 프로젝트의 목표
| 항목 | 목표 |
|------|------|
| 코골이 소리 감소 | **10~15 dB** (체감상 절반 이하로 감소) |
| 말소리 유지 | **50% 이상** (비상시 대화 가능) |
| 반응 속도 | **0.005초 이내** (자연스러운 상쇄) |

### dB(데시벨)이란?
```
소리 크기 비교표:
┌────────────────────────────────────┐
│ 60 dB: 일반 대화                    │
│ 55 dB: 작은 코골이                  │
│ 50 dB: 조용한 사무실                │
│ 45 dB: ← 15dB 감소 후 (목표)        │
│ 40 dB: 도서관                       │
└────────────────────────────────────┘

10~15 dB 감소 = 체감상 "거의 안 들림" 수준
```

## 3. 어떻게 작동하나요?

### 전체 흐름 (5단계)

```
┌──────────────────────────────────────────────────────────────────┐
│                                                                  │
│  [1단계]           [2단계]           [3단계]                     │
│  코골이 감지   →   컴퓨터 분석   →   반대 소리 계산               │
│  (마이크 1개)       (FxLMS)          (적응 필터)                  │
│                                                                  │
│                        ▼                                         │
│                                                                  │
│  [5단계]           [4단계]                                       │
│  실시간 조정   ←   스피커 출력                                   │
│  (오차 마이크)      (반대 소리)                                  │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

### 각 단계 쉬운 설명

#### 1단계: 코골이 감지 (마이크 1개)
```
       ┌─────────┐
       │ 코골이  │
       │  소리   │
       └────┬────┘
            │
            ▼
       ┌───────┐
       │마이크1│   ← 코골이 근처에 배치
       └───────┘
```
- 코골이 소리를 **빨리** 감지해야 합니다
- 마이크 1개를 사용해 코골이 소리를 캡처합니다

#### 2단계: 컴퓨터 분석
```
마이크 신호 → [컴퓨터] → "이 소리의 반대를 만들어야 해!"
```
- 코골이 소리의 **패턴(주파수, 크기, 위상)**을 분석합니다
- 초당 48,000번 소리를 측정합니다
- 100~300Hz 대역의 코골이 소리에 집중합니다

#### 3단계: 반대 소리 계산 (FxLMS 알고리즘)
```
원래 소리:  ⌒⌒⌒  → 계산 → 반대 소리: ⌣⌣⌣
```
- **FxLMS**는 "반대 소리를 얼마나 잘 만들었는지"를 계속 확인하며 개선하는 방법입니다
- 마치 시험을 보고 → 틀린 문제를 확인하고 → 다음엔 더 잘 맞히는 것과 같습니다

#### 4단계: 스피커 출력
```
    ┌───────────────────────────┐
    │         청자 귀           │
    │            ▲              │
    │      ┌─────┼─────┐        │
    │  스피커1        스피커2   │   ← 귀 양쪽에 배치
    │      │           │        │
    │      └─────┬─────┘        │
    │     반대 소리 출력         │
    └───────────────────────────┘
```
- 계산된 반대 소리를 스피커로 출력합니다
- 귀 양쪽에 스피커를 배치해 효과를 높입니다

#### 5단계: 실시간 조정 (오차 마이크)
```
┌────────────────────────────────────────┐
│  귀 옆 마이크: "아직 소리가 남아있네"    │
│        │                               │
│        ▼                               │
│  컴퓨터: "조금 더 조정해야겠다"          │
│        │                               │
│        ▼                               │
│  스피커: 조정된 반대 소리 출력           │
└────────────────────────────────────────┘
```
- 귀 옆에 있는 **오차 마이크**가 남은 소리를 측정합니다
- 컴퓨터가 이 정보로 계속 보정합니다
- 이 과정이 **1초에 수백 번** 반복됩니다

## 5. 소프트웨어 구성

### 파일 구조 (간단 버전)

```
ANC_Bed_Speaker/
└── prototype_student/
    ├── main.py               ← 프로그램 시작점
    ├── anc/                  ← 핵심 소음 제어 코드
    │   ├── fxlms.py          ← 반대 소리 만드는 알고리즘
    │   ├── secondary_path.py ← 스피커→귀 경로 학습
    │   └── dsp.py            ← 신호 처리 도구들
    ├── tools/                ← 분석 도구
    │   └── measure_psd.py    ← 소음 감소 효과 측정
    └── docs/                 ← 설명 문서
```

### 핵심 용어 쉬운 설명

| 용어 | 쉬운 설명 | 비유 |
|------|----------|------|
| **ANC** | 능동 소음 제어 (Active Noise Control) | 소음에 반대 소리를 내서 상쇄 |
| **FxLMS** | 반대 소리를 점점 잘 만드는 학습 방법 | 시험 보고 오답 확인하며 실력 향상 |
| **2차 경로** | 스피커 → 귀까지 소리가 가는 길 | 스피커 소리가 귀에 어떻게 들리는지 |
| **적응 필터** | 계속 조정되는 소리 변환기 | 상황에 맞게 조절되는 스피커 볼륨 |
| **에러 마이크** | 남은 소음을 측정하는 마이크 | 내가 얼마나 잘했는지 확인하는 선생님 |

## 6. 사용 방법

### 3단계로 사용하기

아래 명령은 `prototype_student/` 디렉터리에서 실행합니다.

```
┌─────────────────────────────────────────────────────────────┐
│ [1단계] 경로 학습 (처음 1회)                                 │
│                                                             │
│   $ python main.py learn-secondary --duration 10            │
│                                                             │
│   → 스피커에서 "쉬~" 소리가 10초간 나옴                      │
│   → 이 소리가 귀까지 어떻게 가는지 학습                      │
│   → secondary_path.npz 파일 생성                            │
└─────────────────────────────────────────────────────────────┘
                            ▼
┌─────────────────────────────────────────────────────────────┐
│ [2단계] 기준 측정 (비교용)                                   │
│                                                             │
│   $ python main.py baseline --duration 30                   │
│                                                             │
│   → ANC 끈 상태로 30초간 소리 측정                           │
│   → 나중에 "얼마나 줄었나" 비교할 때 사용                    │
│   → baseline.npz 파일 생성                                  │
└─────────────────────────────────────────────────────────────┘
                            ▼
┌─────────────────────────────────────────────────────────────┐
│ [3단계] 소음 제거 실행                                       │
│                                                             │
│   $ python main.py run --secondary secondary_path.npz       │
│                                                             │
│   → 실시간으로 코골이 소리 감소 시작                         │
│   → Ctrl+C를 누르면 종료                                    │
│   → anc_run.npz 파일에 결과 저장                            │
└─────────────────────────────────────────────────────────────┘
```

### 결과 확인하기

```bash
$ python tools/measure_psd.py --baseline baseline.npz --anc anc_run.npz
```
- 코골이 소리가 얼마나 줄었는지 그래프로 보여줍니다
- 목표: 100~300Hz 대역에서 10~15dB 감소

## 7. 자주 묻는 질문 (FAQ)

### Q1. 귀에 이어폰을 꽂아야 하나요?
**아니요!** 스피커가 귀 옆에 있어서 이어폰 없이 사용합니다.
베개에 스피커를 넣거나 헤드보드에 설치합니다.

### Q2. 모든 소리가 다 없어지나요?
**아니요.** 코골이 소리(100~300Hz의 낮은 음)만 줄입니다.
대화 소리(300Hz 이상)는 잘 들립니다. 비상시 의사소통이 가능합니다.

### Q3. 반응이 얼마나 빠른가요?
**0.005초(5밀리초) 이내**입니다.
눈 깜빡이는 것보다 50배 빠릅니다.

### Q4. 방 전체의 코골이가 없어지나요?
**아니요.** 청자(잠자는 사람) 귀 위치에서만 효과가 있습니다.
방 전체 소음 제거는 이 프로젝트의 목표가 아닙니다.

### Q5. 배터리로 사용할 수 있나요?
현재 버전은 **전원 연결이 필요**합니다.
노트북이나 라즈베리파이에 연결해서 사용합니다.

### Q6. dB가 뭔가요?
데시벨(dB)은 소리 크기를 나타내는 단위입니다.
- 10dB 감소 = 소리가 **절반**으로 느껴짐
- 15dB 감소 = 소리가 **1/3**로 느껴짐

## 8. 작동 원리 더 알아보기 (선택 읽기)

### 파동 상쇄의 과학

소리는 공기의 진동(파동)입니다.

```
소리 파동의 기본:
  
  높음  ╱╲      ╱╲      ← 공기가 밀림
       ╱  ╲    ╱  ╲
      ╱    ╲  ╱    ╲
  ────────────────────────
      ╲    ╱  ╲    ╱
       ╲  ╱    ╲  ╱
  낮음  ╲╱      ╲╱      ← 공기가 당겨짐
```

반대 파동(역위상)을 만들면:
```
원래:       ╱╲      ╱╲
           ╱  ╲    ╱  ╲
  ────────────────────────
           ╲  ╱    ╲  ╱
반대:       ╲╱      ╲╱

합치면:
  ────────────────────────  ← 거의 평평해짐!
```

이것이 **상쇄 간섭(Destructive Interference)**입니다.

### 왜 "적응"이 필요한가?

코골이 소리는 **계속 변합니다**:
- 크기가 변함 (숨 쉴 때마다)
- 패턴이 변함 (자세에 따라)
- 환경이 변함 (이불, 베개 위치)

그래서 반대 소리도 **실시간으로 조정**해야 합니다.
FxLMS 알고리즘이 이 조정을 자동으로 합니다.

```
[코골이 변화] → [FxLMS가 감지] → [반대 소리 조정] → [계속 상쇄]
      │                                                 │
      └──────────── 반복 (1초에 수백 번) ─────────────────┘
```
