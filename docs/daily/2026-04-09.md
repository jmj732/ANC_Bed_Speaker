# 2026-04-09

## 오늘 한 일

- `anc-rt.service`를 `period=64`, `buffer=768`, `start-fill=3`, `xrun-fill=3`, `mb-bands=1`, `adapt-delay=400ms` 운영값으로 복구하고 live 상태를 확인했다.
- latency 분해 기록을 다시 확인했다.
  - `period=64` 기준 SW period alignment는 `1.333ms`
  - `LL IIR` 포함 secondary path total latency 기록은 `192 samples = 4.00ms`
  - `ref mic lead = 0 samples`라서 현재 ANC 감쇠 한계는 SW latency보다 feedforward causality 부족 쪽이 더 크다고 정리했다.
- CPU 2개를 하나처럼 합쳐 쓰는 것은 불가능하고, 현재 구조에서는 `ANC RT loop`와 `audio IRQ/housekeeping` 분리가 맞는 방향이라고 정리했다.
- `fxlms_step()` hot loop에 수동 NEON helper를 추가했다.
  - `S_hat dot x`
  - `w dot x`
  - `fx_pow`, `x_pow` 동시 누산
  - weight update
- `cbuf_push()`의 `%` modulo를 분기형 wrap로 바꿨다.
- `capture-bench` 모드를 추가했다.
  - playback 없이 capture만 열어서 `read`, `avail`, `delay`, `xrun`을 본다.
  - `alsa_init_capture_only()`를 추가하고 playback 없는 recovery 경로도 같이 정리했다.
- 빌드 기본값을 `-O3 -mcpu=cortex-a76`로 올렸다.
- 앞으로 날짜별 작업 기록을 남기기 위해 `docs/daily/` 규칙을 문서화했다.
- `docs/handoff/NEXT_SESSION_PROMPT.md`를 현재 상태 기준으로 전면 갱신했다.
  - `64/768/fill=3/mb=1` 운영값
  - `capture-bench`
  - 마이크-only 전제
  - 다음 우선순위를 capture ALSA/I/O 쪽으로 재정렬

## 구현 메모

- 자동 벡터화만으로는 hot loop가 거의 벡터화되지 않았다.
- 수동 NEON은 latency 자체를 크게 줄이기보다 `compute headroom` 확보가 목적이다.
- 기대 효과는 `fill=2`, `period=48/32`, `mb-bands=2` 재도전용 여유 확보 쪽이다.

## 검증

- `make`
- `cd runtime && ../build/anc sim --tone-hz=82 --band-lo=70 --band-hi=170 --mb-bands=1`
  - 결과: `[SIM 82Hz] reduction=+20.83dB`
- `cd runtime && ../build/anc capture-bench --capture-secs=2 --period=64 --buffer-mult=12`
  - 실행 전 `anc-rt.service`를 잠깐 내려서 ALSA 장치를 비웠고, 실행 후 다시 올렸다.
  - 결과:
    - `[CAP 1.0s]` `read=1.333/1.415 ms`, `avail=0/0/0`, `delay=0/0/0`, `xr=0`
    - `[CAP 2.0s]` `read=1.333/2.670 ms`, `avail=0.1/0/64`, `delay=0.1/0/64`, `xr=0`
  - 셸 직실행이라 `SCHED_FIFO`는 권한 부족 경고가 났다.

실기 `run` 성능 비교는 아직 하지 않았다.

- 스피커를 다시 연결한 뒤 RT `capture-bench`를 재측정했다.
- `period=64`, `buffer=768`에서 3회 모두 `xr=0`, `read_avg=1.333ms`, `avail/delay=0`이었다.
- 해석: 현재 `read_ms`는 copy보다 `period=64` block alignment 성격이 강하다.
- `period=48`은 첫 1초 `read=1.000/1.091ms`, `xr=0`까지 확인했고 중간에 중단됐다.

## ANC A/B 요약

- `fill=3`
  - `xr=0`
  - `compute≈0.070~0.077ms`
  - 같은 배치 재검증에서 divergence/reset 반복
  - best 개선은 `err≈+0.2dB`, `tone≈+0.3dB` 수준
- `fill=2`
  - `xr=0`
  - `compute≈0.074~0.078ms`
  - 약 `43초` 동안 divergence/reset 없음
  - `err`는 baseline 대비 대체로 `+0.0~+0.6dB`
  - `tone best≈+2.5dB`
- 판단: 현재 배치에서는 `fill=2`가 `fill=3`보다 훨씬 안정적이었다.
